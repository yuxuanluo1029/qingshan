# 部署说明

本文采用 **Vercel（前端） + Render（后端）** 方案。

你当前后端已经在 Render，可直接从第 3 节开始。

## 1. 部署前准备

1. 腾讯地图 Key（前端 JS API + WebService API 权限）
2. 腾讯混元服务端密钥：
   - `TENCENT_SECRET_ID`
   - `TENCENT_SECRET_KEY`
3. 阿里云 DashScope API Key（若使用通义）
   - `ALIYUN_API_KEY`
4. Git 仓库可访问

## 2. 后端部署（Render）

目录：`backend`

### 2.1 使用 render.yaml

仓库内已提供：`backend/render.yaml`

### 2.2 Render 服务配置

- Runtime: Node
- Root Directory: `backend`
- Build Command: `npm install`
- Start Command: `npm run start`

### 2.3 环境变量

必须配置：
- `PORT=3000`
- `CORS_ORIGIN=https://你的前端域名`
- `AI_PROVIDER=aliyun`（或 `auto`）
- `ALIYUN_API_KEY=...`
- `ALIYUN_MODEL=qwen-plus`
- `ALIYUN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1`
- `TENCENT_SECRET_ID=...`
- `TENCENT_SECRET_KEY=...`
- `TENCENT_HUNYUAN_REGION=ap-guangzhou`
- `TENCENT_HUNYUAN_MODEL=hunyuan-turbos-latest`

部署成功后，记录后端地址，例如：
`https://chengji-backend.onrender.com`

## 3. 前端部署（Vercel）

目录：`frontend`

仓库内已提供：`frontend/vercel.json`

### 3.1 Vercel 项目设置

- Framework Preset: Vite
- Root Directory: `frontend`
- Build Command: `npm run build`
- Output Directory: `dist`

### 3.2 前端环境变量

- `VITE_TENCENT_MAP_KEY=...`
- `VITE_API_BASE_URL=https://chengji-backend.onrender.com`

说明：前端现已支持 `VITE_API_BASE_URL`，会自动请求 `https://你的后端域名/api/*`。

部署成功后会生成前端网址，例如：
`https://chengji.vercel.app`

## 4. 腾讯地图白名单配置

前往腾讯位置服务控制台，把以下域名加入白名单（Key 安全设置）：
1. Vercel 前端域名（如 `chengji.vercel.app`）
2. 自定义域名（如有）
3. 本地调试域名（`localhost`）

同时确认该 Key 已启用：
1. JavaScript GL API（地图展示）
2. WebService API（地点搜索/地理编码）

## 5. 联调检查清单

1. 打开前端首页是否正常
2. 进入路线页地图是否可见
3. 聊天页是否能获得后端回复（非 mock）
4. 节点页问答是否可返回
5. 报告页是否可完整展示
6. 访问 `https://你的后端域名/health`，确认 `providers.aliyun` 为 `ready`
