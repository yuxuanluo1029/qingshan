# 部署说明

本文采用 **Vercel（前端） + Render（后端）** 方案。

## 1. 部署前准备

1. 腾讯地图前端 Key（并配置域名白名单）
2. 腾讯混元服务端密钥：
   - `TENCENT_SECRET_ID`
   - `TENCENT_SECRET_KEY`
3. Git 仓库可访问

## 2. 后端部署（Render）

目录：`backend`

### 2.1 使用 render.yaml

仓库内已提供：`backend/render.yaml`

### 2.2 Render 服务配置

- Runtime: Node
- Root Directory: `backend`
- Build Command: `npm install`
- Start Command: `npm run start`

### 2.3 环境变量

必须配置：
- `PORT=3000`
- `CORS_ORIGIN=https://你的前端域名`
- `TENCENT_SECRET_ID=...`
- `TENCENT_SECRET_KEY=...`
- `TENCENT_HUNYUAN_REGION=ap-guangzhou`
- `TENCENT_HUNYUAN_MODEL=hunyuan-turbos-latest`

部署成功后，记录后端地址，例如：
`https://chengji-backend.onrender.com`

## 3. 前端部署（Vercel）

目录：`frontend`

仓库内已提供：`frontend/vercel.json`

### 3.1 Vercel 项目设置

- Framework Preset: Vite
- Root Directory: `frontend`
- Build Command: `npm run build`
- Output Directory: `dist`

### 3.2 前端环境变量

- `VITE_TENCENT_MAP_KEY=...`
- `VITE_API_BASE_URL=https://chengji-backend.onrender.com`

部署成功后会生成前端网址，例如：
`https://chengji.vercel.app`

## 4. 腾讯地图白名单配置

前往腾讯位置服务控制台，把以下域名加入白名单：
1. Vercel 前端域名（如 `chengji.vercel.app`）
2. 自定义域名（如有）
3. 本地调试域名（`localhost`）

## 5. 联调检查清单

1. 打开前端首页是否正常
2. 进入路线页地图是否可见
3. 聊天页是否能获得后端回复
4. 节点页问答是否可返回
5. 报告页是否可完整展示

## 6. 本地命令回顾

后端：
```bash
cd E:\青衫竞渡\backend
cp .env.example .env
npm install
npm run dev
```

前端：
```bash
cd E:\青衫竞渡\frontend
cp .env.example .env
npm install
npm run dev
```
