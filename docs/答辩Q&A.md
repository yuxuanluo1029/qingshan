# 答辩 Q&A

## Q1：你的项目和普通旅游导览有什么区别？

A：我们不是“景点介绍”，而是“研学任务驱动”的学习系统。用户会得到可执行观察任务、结构化知识点和最终学习报告，强调教育价值而非纯浏览。

## Q2：为什么说它符合“智能教育与文化”赛道？

A：系统核心围绕历史文化知识讲解、城市文脉理解、任务化学习与学习反馈，目标是支持课程化应用与公共文化教育，完全对齐赛道方向。

## Q3：AI 在这里的真实价值是什么？

A：AI 不只是回答问题，而是动态生成讲解结构、现场观察方法、路线说明和学习建议，降低用户从“看到”到“理解”的门槛。

## Q4：腾讯地图在系统里承担什么角色？

A：腾讯地图用于点位可视化、路径辅助与空间语境构建。它让学习从抽象文本回到真实城市现场，实现“地理空间 + 历史文化”的融合。

## Q5：如果没有配置大模型密钥，系统还能演示吗？

A：可以。后端内置 mock 回退，保证无密钥场景也能完整演示流程；配置腾讯混元密钥后自动切换到真实模型。

## Q6：你们如何保证敏感信息安全？

A：混元密钥只在后端环境变量中使用，前端不保存密钥。前端仅调用后端 API，符合密钥隔离原则。

## Q7：系统有哪些可扩展方向？

A：
1. 接入更多城市与专题课程模板
2. 增加教师端（任务配置、班级管理、作业评估）
3. 加入语音讲解与多模态识别（图像/文物识别）
4. 报告自动导出为海报或课程简报

## Q8：比赛现场如果网络不稳定怎么办？

A：
1. 前端页面和本地 mock 能完成主要演示
2. 地图和模型失败时有明确降级提示
3. 可提前录制关键路径演示视频作为兜底
