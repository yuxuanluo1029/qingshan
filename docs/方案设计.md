# 方案设计

## 1. 作品名称

城迹：城市历史文化研学智能体

## 2. 应用场景

1. 高校与中学城市研学课程
2. 博物馆、历史街区、纪念馆导览活动
3. 城市文化传播与公众教育项目
4. 比赛现场答辩中的系统功能演示

## 3. 设计理念

1. 从“旅游打卡”升级为“证据驱动的城市学习”
2. 从“被动看景”升级为“任务驱动观察 + AI 辅助解释”
3. 从“一次体验”升级为“可复盘的学习报告”

核心理念：
- 历史：以时间线和事实为骨架
- 文化：以人物、民俗、非遗和空间叙事为内容
- 城市：以真实场景作为学习现场

## 4. 功能设计

### 4.1 研学设置

- 输入城市、时长、兴趣主题、学习偏好
- 自动生成个性化研学路线

### 4.2 AI 导师对话

- 提供路线说明与策略建议
- 支持实时提问（背景、证据、讲解提纲）
- 提供现场观察建议与表达结构

### 4.3 路线与地图

- 腾讯地图展示点位与顺序
- 每个点位提供：时长、推荐理由、学习要点

### 4.4 节点研学任务

- 历史文化简介
- 知识亮点
- 观察任务/文化问答/田野记录
- 拍照记录建议

### 4.5 学习画像与报告

- 输出学习关键词、关键亮点、能力画像
- 给出下一步深挖建议

## 5. 技术方案

### 5.1 前端

- React + Vite + TypeScript
- 组件化页面：Index / Setup / Chat / Route / Node / Bond / Report
- Framer Motion 提供页面动效
- 腾讯地图 GL API 实现地图与点位展示

### 5.2 后端

- Node.js + Express
- API：
  - `/api/ai/chat`
  - `/api/ai/knowledge`
  - `/api/ai/route`
- 腾讯混元仅在后端调用，避免前端暴露密钥
- 无密钥场景支持 mock 回退，确保系统可演示

### 5.3 数据层

- 城市、点位、导师、路线、任务、报告模板统一在 `mockData.ts`
- 支持多城市（杭州、成都、长沙、西安、武汉）
- 支持多兴趣标签与多时长策略

## 6. 系统架构

```text
用户
  ↓
React 前端（页面交互 + 地图展示 + 状态管理）
  ↓ HTTP
Express 后端（业务接口 + 模型网关）
  ↓ SDK
腾讯混元模型
```

## 7. 创新点

1. 将“城市空间”转化为“教育现场”，实现可执行任务闭环
2. AI 输出不止问答，还给“可讲解结构”和“观察方法”
3. 路线、地图、任务、报告四者联动，适合比赛现场完整演示
4. 支持无密钥 fallback，提升项目演示稳定性

## 8. 预期效果

1. 用户可在短时间内形成结构化历史文化理解
2. 教师或讲解者可直接复用输出作为课程/导览素材
3. 比赛答辩可展示“方案—实现—演示—报告”全链路能力
